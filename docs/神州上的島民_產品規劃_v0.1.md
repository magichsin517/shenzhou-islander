## E. 命名與品牌定位

### 名字風險評估

| 維度 | 「神州上的島民」 | 評估 |
|------|------------------|------|
| 識別度 | 極高（一聽就懂、有畫面） | ✅ |
| 自嘲感 / 歸屬感 | 強（被叫的詞自己拿回來用） | ✅ |
| 台灣 App Store 審查 | 無問題 | ✅ |
| 大陸應用商店審查 | 「島民」+「神州」對立感太強，過審風險高 | ⚠️ |
| 商標查詢 | 待查（中文 + 英文 Mainlander / Islander） | ⏳ |
| SEO / 搜尋 | 不易被誤搜，獨特 | ✅ |
| 對非用戶解釋成本 | 0（一句話講完） | ✅ |

**結論**：在「只上 App Store 台灣區 + Web」前提下，名字可用，是強資產。
**但備案要先有**——萬一未來想雙端，得切換。

### 命名備案（3 個）

| 名字 | 調性 | 優缺 |
|------|------|------|
| **島民日常 / Islander Daily** | 中性、可上雙端 | 弱化反叛感、像生活雜誌 |
| **過海 / Crossing** | 文藝、隱喻 | 太隱晦、新人看不懂 |
| **海漂 / Drifters** | 對位「滬漂」+「島民」 | 跟現有「滬漂」群混淆 |

**永興建議路線**：
- 主品牌用「**神州上的島民**」（中文圈內傳播）
- 英文名用「**Islander**」或「**Mainlander**」（中性、上架友善）
- 註冊一個無敏感字的母公司 / 商標：例如「島民工作室」

### 品牌調性

| 維度 | 定位 |
|------|------|
| 性格 | 像「住了 10 年的老島民，正在替剛來的新人指路」——溫暖、不裝、帶一點自嘲 |
| 不是 | 不是「攻略網站」（冷）、不是「同鄉會」（老）、不是「文化交流」（虛） |
| 語言 | 繁體為主、簡繁混用無所謂、可以用兩岸都懂的梗 |
| 視覺方向 | 海島 + 大陸的對位（不是政治符號，是地理感）；暖色為主；手繪 / 不銳利 |
| 用色提案 | 海水藍 #3A7FB8（島）+ 沙土黃 #D4A574（神州）+ 米白底 #FAF6EF |

### Slogan 候選

1. **「在這座島之外，還有一座島」** ←推薦
2. 「找到你的老島民」
3. 「不是滬漂，是島民」
4. 「離家很遠，但有人懂」

## A. Onboarding 三問 + 島民卡

### 三問的具體選項

每一問都是**單選題**（降低門檻），但每一問都會生成一個「永久標籤」掛在用戶資料上。

#### 問題 1：你為什麼來？

```
🎒 來玩（短期旅遊 / 出差 / 探親 / 看演唱會）
🏠 來生活（陪伴 / 婚姻 / 退休 / 探索）
💼 來工作（外派 / 求職 / 創業）
🎓 來唸書（交換 / 攻讀學位）
🤔 還在想 / 不確定
```

→ **觸發分支**：
- 選「來玩」→ 後續路徑簡化，主推「攻略 + 線下活動」
- 選「來生活/工作/唸書」→ 走完整 onboarding，進入「社群核心」
- 選「還在想」→ 推一篇「滬漂前要知道的 10 件事」（你 Podcast 內容可改編）

#### 問題 2：你現在的角色？

```
🎓 學生（交換生 / 研究生 / 大學）
💼 上班族（外商 / 中資 / 國企 / 創業公司）
🚀 創業者 / 自由業
🏠 陪伴者（陪配偶 / 陪子女 / 陪父母）
🌿 退休 / Gap / 在思考下一步
🛬 還沒來，準備中
```

→ 這一題決定**「先看到誰的內容」**——學生會看到學生內容，創業者看到創業者內容。

#### 問題 3：你來多久了？

```
⏳ 還沒來（計劃中 / 正在辦手續）
🌱 0–6 個月（新島民）
🌿 6 個月–2 年（熟悉中）
🌳 2–5 年（在地化中）
🌲 5–10 年（深度滬漂）
🏔 10 年以上（老島民）
```

→ 這一題是**社群的命脈**：自動決定誰是「mentor」、誰是「mentee」。
→ **5 年以上的人**會被系統標記為「**老島民**」，享有：
- 在答疑頁面被優先 @
- 可解鎖「島民導師」徽章
- 線下活動優先報名席位

### 島民卡（Islander Card）設計

三問結束後，**幫用戶自動生成一張卡片**，這是 app 的核心人設物件。

```
┌─────────────────────────────────┐
│  🌳 老島民 · 第 1,234 號         │
│                                  │
│  小美 · @meimei                  │
│  ───                             │
│  💼 設計師 · 來工作               │
│  🏔 在上海 7 年 · 老島民          │
│  📍 徐匯區 / 常去：襄陽南路       │
│                                  │
│  「正在學上海話，求飯友」          │
│                                  │
│  [認識小美]  [私訊]               │
└─────────────────────────────────┘
```

#### 卡片欄位（MVP）

| 欄位 | 資料來源 | 必填 |
|------|---------|------|
| 暱稱 + 編號 | 用戶輸入 + 系統發號 | ✅ |
| 階段徽章（新島民 / 老島民 / ...） | 三問計算 | ✅ |
| 為什麼來 + 現在角色 | 三問 | ✅ |
| 來多久（自動更新年數） | 入境日期 + 系統時鐘 | ✅ |
| 一句話自介（限 30 字） | 用戶輸入 | ✅ |
| 常駐區（區級，不到地址） | 用戶選 | 推薦 |
| 常去地點（最多 3 個） | 用戶選 / 輸入 | 選填 |
| 興趣 tag（最多 5 個） | 用戶選 | 選填 |
| 頭像 | 用戶上傳 / 預設 | ✅ |

#### 卡片背面（深度資訊，點開才看到）

- 已回答的問題數 / 被感謝數
- 來自台灣哪裡（縣市，可隱藏）
- 「我最想推薦給新島民的 3 件事」（自填）
- 「我在上海最常去吃的店」（自填，可導到 google maps / dianping）

### 為什麼這個架構好？

1. **3 個問題 = 3 個標籤系統**：身份、階段、來意。任何後續功能都可以基於這三個維度做個性化
2. **自動 mentor / mentee 配對**：5 年+ → 6 個月以下，系統可主動推薦關係
3. **「來多久」會自動長**：用戶留得越久，徽章越高，**沉沒成本變留存動力**
4. **新人看到的不是空頁面**：onboarding 結束立刻給看「跟你像的 5 個人 + 跟你不像但能幫你的 3 個人」

## D. 身份驗證策略

### 問題拆解

| 風險 | 嚴重程度 |
|------|---------|
| 大陸用戶冒充台胞混入 | 高（毀社群調性） |
| 商家 / 仲介偽裝引流 | 中（騷擾用戶） |
| 機器人 / 假帳號 | 中 |
| 驗證太重嚇跑用戶 | **致命**（社群冷啟動最大殺手） |

### 三層漸進信任模型

**核心原則**：**進得來 ≠ 信得過。讓信任變成可累積的資產。**

| 層級 | 進入門檻 | 能做什麼 | 限制 |
|------|---------|---------|------|
| **🌱 訪客** | 註冊（Email / Apple ID） | 瀏覽公開內容、看島民卡 | 不能發言、不能私訊 |
| **🌿 島民** | 通過「島民識別」（見下） | 發問、回答、報名活動 | 私訊每日上限 3 人 |
| **🌳 老島民** | 來 2 年+ 且被感謝 10 次+ | 全部功能、可背書新人 | — |

### 「島民識別」的軟驗證怎麼做？

**不要**直接要台胞證 / 護照——隱私風險、用戶反感。
**改用「組合題式驗證」**：

#### 方案 A：三選一（任一達成即升級為島民）

1. **邀請碼**：被現有島民邀請（每人月配 3 個碼，質量管控）
2. **內容驗證**：上傳一張「能證明你在台灣 / 在上海的物件」，例如：
   - 悠遊卡照片 / 統一發票照片 / 台胞證封面（遮住敏感資訊）
   - 上海公交卡 / 居住證封面（遮住）
   - **不存原圖，只做人工 / AI 一次性審核**
3. **問答驗證**：回答 3 道「台胞才會懂」的題目（不是政治題，是生活題）
   - 例：「鼎泰豐的招牌料理是？」「上海話『鄉下人』怎麼說？」
   - 全對才通過，可重答（避免不熟悉的人猜中）

#### 方案 B：邀請制 + 老島民背書

- **冷啟動期（前 1000 人）只走邀請制**：你 + Seed 100 人手動邀
- **過了冷啟動才開放方案 A**

**建議組合**：
- Phase 1（0-100 人）：純邀請制，你親發碼
- Phase 2（100-1000 人）：邀請碼 + 內容驗證
- Phase 3（1000+）：開放問答驗證 + 邀請碼

### 退場機制

| 行為 | 後果 |
|------|------|
| 被檢舉 3 次（不同人） | 自動降級到訪客，需要重新驗證 |
| 被檢舉 10 次 | 永久封禁 |
| 發政治敏感內容 | 該則隱藏，發 3 次降級 |
| 商業推廣（未經許可） | 該則刪除，發 2 次封禁 |

### 政治敏感內容紅線（社群守則 v0）

**這是台胞社群最大內爆點，要先寫死**：

1. 不討論兩岸政治立場（藍綠 / 統獨 / 選舉）
2. 不討論大陸內政（不評論政策、不踩線）
3. 不使用對立性詞彙（「老共」「台獨」「皇民」「支那」等）
4. 文化、歷史討論可以，**評論性政治立場不行**
5. **這不是審查，是讓不同立場的人都能在這裡好好生活**

→ Onboarding 結束前彈一頁讓用戶閱讀 + 勾選「我同意」。
→ 違規的處理透明公開（不是黑箱封號）。

## B. 場景 A 島民答疑 PRD

### 問題陳述

新島民來上海的前 6 個月，平均要問 **30+ 個「該怎麼辦」的問題**：
- 怎麼辦居住證？哪個區好住？
- 哪裡看牙醫？台胞健保能用嗎？
- 怎麼開銀行戶？怎麼用支付寶？
- 哪裡買得到台灣牌子的東西？

這些問題的答案散落在：
- 微信群（找不到、會洗版）
- 小紅書（演算法決定看到誰）
- 豆瓣老貼（介面爛、過時）
- 老島民朋友（不一定剛好認識）

**機會**：把「老島民腦子裡的答案」結構化沉澱下來，**讓答過一次的問題不用再答**。

### 核心使用者故事

> **作為一個來上海 2 個月的新島民，**
> **我想問「在徐匯區辦居住證最快的派出所是哪間？」，**
> **而我希望 6 小時內有住徐匯 3 年以上的老島民來回答我。**

### MVP 功能清單

#### 1. 發問流程

```
[+ 發問]
  ↓
分類：辦事 / 住 / 吃 / 工作 / 健康 / 教育 / 其他
  ↓
標題 + 補充說明（限 500 字）
  ↓
位置（區）+ 緊急程度（隨便問問 / 等回答 / 很急）
  ↓
[發布]
```

#### 2. 自動 @ 邏輯（這是命脈）

發問時，**系統自動找 3 個最可能回答的人**並通知：

| 因素 | 權重 |
|------|------|
| 在該分類下回答過 → 標籤匹配 | 40% |
| 來上海年數 ≥ 2 年 → 老島民 | 30% |
| 常駐區跟發問者區一致 | 20% |
| 最近 7 天活躍 | 10% |

**重要**：不是強制 @，是「邀請回答」——被 @ 的人收到 push「有個人需要你」，可以選擇答或不答。

#### 3. 回答機制

- **答案質量分級**：
  - 簡短回應（< 50 字）→ 顯示為「補充」
  - 完整答案（≥ 50 字）→ 顯示為「主答案」
  - 主答案可被其他人「也是這樣」（按讚）
- **獎勵**：
  - 被選為最佳答案 → +20 信任點
  - 被感謝 → +5 信任點
  - 累計 50 點解鎖「島民導師」徽章

#### 4. 沉澱為「島民百科」

問題被回答 3 次以上 + 標題被點擊 100 次以上 → **自動進「島民百科」**

| 機制 | 細節 |
|------|------|
| 內容歸屬 | 原回答者 + 補充者，類似 wiki 但有貢獻者標示 |
| 編輯權 | 老島民可申請編輯權，編輯紀錄公開 |
| 搜尋 | 全文搜尋 + 標籤搜尋 + 位置搜尋 |
| SEO | Web 版開放搜尋引擎抓取 → 帶外部流量 |

#### 5. 過時提醒

「居住證辦理流程」這種事每幾年會變。**每篇答案上線 1 年後**：
- 自動標記「需要驗證」
- 推送給原作者 + 老島民「這篇還對嗎？」

### 不做（明確排除）

- ❌ AI 自動回答（這是社群，不是 chatbot）
- ❌ 商家入駐（避免淪為點評）
- ❌ 直播 / 影片（PMF 還沒驗證，先做文字）
- ❌ 私訊（Phase 2 再開）

### 成功指標

| 指標 | 目標（前 3 個月） |
|------|----------------|
| 平均回答時間 | < 6 小時 |
| 「有效回答率」（有≥1 個 ≥50 字答案） | > 80% |
| 老島民週活躍率 | > 40% |
| 新島民第 7 天留存 | > 50% |
| 進「島民百科」的問題數 | > 100 篇 |

### 反指標（出現要警惕）

- 平均回答時間 > 24 小時 → 老島民流失或沒被觸發到
- 問題被 0 人回答的比例 > 20% → 用戶會放棄發問
- 重複問題比例 > 40% → 搜尋 / 推薦不好用

## 跨主題：成功指標 + 風險

### 北極星指標（單一）

**「新島民第 30 天有效連結數 ≥ 3」**
- 「有效連結」= 至少有一次雙向互動（互相回答 / 互相報名同個活動 / 互發訊息 ≥ 3 次）

### 護城河來源

| 護城河 | 為什麼別人難複製 |
|--------|----------------|
| 創辦人即用戶 | 永興本人是滬漂 15 年 + Podcast 主持 + 台灣同鄉會出身 |
| 冷啟動人脈 | 0 到 100 的種子用戶來自永興私交，不是買來的 |
| 內容資產 | 島民百科一旦有 500 篇，後來者重做要 2-3 年 |
| 跨業務協同 | 初唤（餐飲）+ 神州島民（社群）兩端都是同一群人，可互相導流 |

### 風險清單

| 風險 | 程度 | 緩解 |
|------|------|------|
| 社群冷啟動失敗 | 高 | 嚴格控制前 100 人質量，寧慢勿濫 |
| 政治內容引爆 | 高 | 守則寫死 + 嚴格執行 + 公開處理 |
| 變現困難 | 中 | Phase 1 不變現，Phase 2 走「線下活動 + 老島民課程」 |
| 對岸找麻煩 | 中 | 只上台灣 App Store + 不在大陸主動推廣 |
| 商標問題 | 低 | 同步註冊「島民工作室」備案 |
| 跟初唤精力衝突 | 中 | 共用人脈但獨立團隊 / 顧問接 |

### 變現路徑（Phase 2 想，現在別動）

1. **線下活動門票**：島民飯局 / 主題沙龍（單場 200-500 RMB）
2. **島民導師付費 1:1**：老島民賣 30 分鐘諮詢（你抽 20%）
3. **企業會員**：在上海招台籍員工的公司付月費接觸島民池
4. **品牌植入**：跟你信任的台灣品牌 / 上海台胞愛用品牌合作（不是廣告位，是策展）

## 下一步

### 7 天內

1. **跟 5 個真實滬漂台胞訪談**（最好你 Podcast 受訪過的）：
   - 問題：「你來上海前 3 個月，最想找誰問話？」「你會用一個 app 來解決這個嗎？」
   - 目標：驗證「找到能講話的人」這個需求強度
2. **跑 onboarding demo**（這份規劃配套的 HTML）給 5 個人看，記錄反應
3. **盤點冷啟動人脈池**：列出你可以親自邀的前 30 人是誰

### 30 天內

1. 找一個技術合夥人 / 接案開發者，估算 MVP 成本
2. 開「島民工作室」（或類似中性名）的公司主體 / 商標
3. Podcast 出一集「我在做一個給滬漂台胞的 app」做預熱（你的訂閱者就是首批用戶）
4. 寫完整 PRD（這份是骨架，要長到 30 頁）

### 90 天內
